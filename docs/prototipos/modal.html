Variações de modais da tela de Projetos

Base de usabilidade para manter toda a operação de projeto concentrada na mesma tela: criar, gerenciar, adicionar membros, alterar papéis e confirmar ações sensíveis sem abrir novas rotas desnecessárias.

Tela central de Projetos

Ideia central: a tela lista os projetos e os modais resolvem as ações pontuais. Isso reduz rotas, mantém contexto e simplifica o MVP.

Escopo Fácil

Projeto principal do PI. Usuário atual é admin e pode gerenciar membros e dados gerais.

Admin

Portal Clínico

Projeto em que o usuário participa como membro. Pode consultar e trabalhar, sem gerenciar pessoas.

Membro

Variações necessárias

Essas são as variações mínimas para a tela de projetos não depender de uma rota separada de configuração.

1. Criar projeto

Modal curto para criar projeto com nome e descrição.

2. Gerenciar projeto

Drawer/modal grande com abas: Geral, Membros e Permissões.

3. Adicionar membro

Modal curto com e-mail e papel inicial no projeto.

4. Alterar papel

Confirmação leve para promover membro a admin ou rebaixar admin.

5. Sair do projeto

Confirmação de saída para membro comum.

6. Arquivar projeto

Ação sensível para admin. Melhor tratar com modal de risco.
